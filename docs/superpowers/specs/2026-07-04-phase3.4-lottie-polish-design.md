# ShotMaster 阶段 3.4 Lottie + 打磨 设计规格

- **日期**：2026-07-04
- **状态**：待审阅
- **分支**：`phase3.4-lottie-polish`
- **前置**：Phase 1–3.3 已合并 `master`（`057fa82`）。`lottie-react` 已安装（Phase 1 deps）。

## 1. 背景与目标

Phase 2 的评分揭晓用 framer-motion 实现（径向光 + 星 spring + 滚动 + 环 + 朱线），效果良好。但通关奖励弹窗（RewardModal）仍用 CSS `animate-star-pop`/`animate-bounce-in`，不够精致。另外，初始 JS 包 ~1.1MB（build 警告），且 a11y 未审计。

**目标**：
1. **Lottie 关键时刻**：RewardModal 庆祝动画 + 3 星完美时 score-reveal 叠加 confetti。
2. **性能 top-3**：路由 code-split + 图片懒加载 + Lottie 按需加载。
3. **可访问性 top-3**：键盘导航 + ARIA/语义 + 色彩对比审计。
4. `prefers-reduced-motion` 下 Lottie 不播放。

## 2. Lottie 关键时刻

### 2.1 RewardModal 庆祝（GameComponents.tsx RewardModal）
- 用 `lottie-react` 的 `<Lottie>` 组件替换 CSS `animate-star-pop`/`animate-bounce-in`。
- 动画内容：confetti 爆发 + 星星弹跳（从 LottieFiles 免费素材获取 JSON）。
- **按需加载**：`const Lottie = React.lazy(() => import('lottie-react'))`；`<Suspense fallback={null}>`。Lottie JSON 文件动态 `import()` 不进初始包。
- `prefers-reduced-motion`：不渲染 Lottie（CSS fallback：静态星级展示）。
- 保留 RewardModal 的 XP/星数/连胜统计 + 按钮（仅替换星动画部分）。

### 2.2 Score-reveal 3 星完美叠加（ScoreResultView.tsx）
- Phase 2 的 framer-motion 揭晓保留。
- 当 `score.stars === 3`（完美通关）时，叠加一个 Lottie confetti overlay（全屏，pointer-events-none，播放一次）。
- 同样按需加载 + reduced-motion 跳过。
- 其他星级（1–2 星）不叠加（framer-motion 揭晓足够）。

### 2.3 Lottie 资源
- 从 LottieFiles 免费下载 2 个 JSON：`confetti-celebration.json`（RewardModal）+ `confetti-burst.json`（score-reveal 3星）。
- 存放：`src/assets/lottie/`。
- 如果无法下载（网络限制），用 `@lottiefiles/dotlottie-react` 或内联一个简单 confetti 动画 JSON。

## 3. 性能 top-3

### 3.1 路由 code-split
- `src/App.tsx`：用 `React.lazy()` 包装页面组件（LevelMap/Learn/Gallery/Community/Profile/LevelDetail/Shoot/Score/Login/Preferences/admin）。
- `<Suspense fallback={<PageSkeleton />}>` 包裹路由。
- `PageSkeleton`：一个极简骨架屏（jp-minimal，washi 底 + 灰块 shimmer）。
- 预期：初始包从 ~1.1MB 降至 ~300–500KB（首屏只加载首页 + 框架）。

### 3.2 图片懒加载
- 全站 `<img>` 加 `loading="lazy"`（Unsplash 参考图、课程缩略图、图库照片）。
- hero 图（首屏可见）不加 lazy（`loading="eager"`）。

### 3.3 Lottie 按需加载
- Lottie JSON 文件用动态 `import()` 加载（`import('../../assets/lottie/confetti.json')`）。
- `lottie-react` 组件本身用 `React.lazy`（不进初始包）。

## 4. 可访问性 top-3

### 4.1 键盘导航
- 路由切换后 focus 主内容（`<main>` `tabIndex={-1}` + `useEffect` focus on route change）。
- 确保所有 interactive 元素（按钮/链接/滑条）键盘可达（Radix Slider 已有；nav 链接已有；补 Skip-to-content 链接）。

### 4.2 ARIA + 语义
- 审计全站：补缺失的 `aria-label`（图标按钮）、`role`（自定义组件）、`alt`（图片）。
- 重点：TopBar/Sidebar nav items、HeroBack 按钮、InteractiveLesson 控件。

### 4.3 色彩对比
- 审计 jp-minimal 令牌：`ink-muted #8A8A86` on `surface #F7F6F2` → 对比比 ~3.5:1（**低于 WCAG AA 4.5:1**，需加深到 ~`#6B6B66`）。
- `accent #B14A3A` on white → ~5.2:1（达标）。
- `gold #9A7B3A` on white → ~3.8:1（小文字不达标；大文字/图标 OK）。
- 修复：`ink-muted` 加深至 `#6B6B66`（AA 达标）；`gold` 用于 ≥18px 或加粗（已满足）。

## 5. 范围
- Lottie RewardModal + score-reveal 3星叠加。
- 路由 code-split + 图片 lazy + Lottie 按需。
- 键盘 focus + ARIA 审计 + 对比修复。
- 不含：admin ④、快门/构图概念、全站 a11y 重审（top-3 聚焦）。

## 6. 验收
1. RewardModal 用 Lottie 庆祝（reduced-motion 下静态 fallback）。
2. score-reveal 3 星叠加 confetti。
3. 路由 code-split：初始包 < 600KB（Vite build 输出）。
4. 图片 `loading="lazy"`（hero 除外）。
5. 路由切换 focus 主内容 + Skip-to-content。
6. `ink-muted` 对比 AA 达标。
7. `npm run check`/`build` 绿；功能不回归。
